Guiding principles

  • Name = visual. A "table" looks like a rendered surface; a "seat" feels like a place, not a slot in an array.
  • Continuity of place. Seats stay put through FORMING → PLAYING → COMPLETED. Only the center changes.
  • One paradigm, many games. Shell owns layout from meta.tableArchetype + maxPlayers; games fill the center and per-seat slots.
  • Relative POV. You at bottom (sit-down) or near-end (head-to-head). Data stays absolute; only the render rotates.
  • Spectators are visible. Cluster-badge on the rim with click-to-expand avatars — never invisible integers, never over the play surface.
  • Tournaments aren't special. Same table + seats. Tournament context lives in a sidebar card; no bespoke chrome.
