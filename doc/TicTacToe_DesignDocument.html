1 Design Principles
The guiding values behind every design decision in this document.
Visual personality
Clean, modern, minimal, and professional. The game is accessible to all ages but the aesthetic is adult — no bright primary-color cartoon styling. Every UI element earns its place.
Color carries meaning
Blue is always X. Teal is always O. Amber signals wins and warnings. Red is reserved for errors and forfeit. Color is never decorative — it communicates state.
Balanced density
Comfortable reading with moderate spacing. Not so sparse it feels empty, not so dense it feels overwhelming. 1.6 line-height on body, 16px base type, generous touch targets.
Mobile is play-first
On mobile the experience is focused entirely on playing. Management, admin, and analytics screens are desktop-only. The mobile UI has three screens: Play, Stats, Settings.
Type does the heavy lifting
Inter Tight's condensed proportions give scores and marks authority. Inter keeps body text relaxed. Monospace (JetBrains Mono) is reserved strictly for technical content — logs, IDs, invite URLs.
Accessibility
WCAG AA contrast minimum throughout. Touch targets ≥44×44px on mobile. X and O marks are distinguishable by color and shape. Sound effects have a mute toggle always accessible.

5 Layout & Navigation
Desktop uses a persistent split layout. Mobile uses a focused bottom-tab navigation.
Desktop (≥768px)
Top navigation bar: logo, nav links (Play, Leaderboard, Stats), theme toggle, username, avatar
Persistent left sidebar (188px): Play, My Stats, Leaderboard, History, Settings, Admin section
Main content area fills remaining width
Admin sidebar section: amber-tinted, shows AI Dashboard + Log Viewer links (admin role only)
Full suite of screens available: all 8 wireframes
Mobile (<768px)
Simplified top bar: logo, theme toggle, avatar only. No nav links.
Bottom tab nav (3 tabs): Play · Stats · Settings
Available screens: Mode selection, Game board, Room creation/join, Personal stats, Settings
Desktop-only screens (Leaderboard, History, AI Dashboard, Log Viewer) show an "open in full browser" prompt
Minimum supported width: 320px. All touch targets ≥44×44px.

Level pills are toggle-on/off in their semantic colour. DEBUG is off by default — too verbose for most sessions. Each active level is highlighted.
Rows expand inline to show the full JSON meta payload in a dark monospace block. No modal, no navigation — stays in context.
Left border colour encodes severity at a glance: blue INFO, amber WARN, red ERROR, deep red FATAL (+ background tint).
Live tail streams new entries via Socket.io, pausing automatically when the user scrolls up to read. Export respects all active filters.
14 Responsive Strategy
Screen-by-screen availability across desktop and mobile breakpoints.
Screen Desktop ≥768px Mobile <768px Mobile notes
Mode selection✓✓Full-width cards, stacked layout
Game board (PvP & PvAI)✓✓Centered board, scores + actions below; no right info panel
Room creation & join✓✓Create + waiting flow only; active rooms list desktop only
Account & profile✓✓Mobile: 4 KPIs + recent games only; full analysis desktop only
Settings✓✓Mobile: Appearance, Sound, Gameplay only
Leaderboard✓✗"Open in full browser" prompt on mobile
Game history✓✗"Open in full browser" prompt on mobile
AI Dashboard✓✗Admin only + desktop only
Log Viewer✓✗Admin only + desktop only
Breakpoint: 768px. Below this threshold the sidebar is hidden, the top nav is simplified to logo + theme + avatar only, and navigation switches to a 3-tab bottom bar (Play · Stats · Settings). The minimum supported width is 320px. All touch targets are ≥44×44px per WCAG AA guidelines.
